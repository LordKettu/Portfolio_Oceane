Sujet du projet

SAE - Développement d'un jeu de plateforme

Dans cette SAE, vous allez mettre à profit les connaissances que vous devez avoir acquises en R5.A.15. Vous devrez représenter un jeu de plateforme, avec un personnage qui évoluera dans ce niveau, rencontrera des ennemis et atteindra finalement la zone d'arrivée.

Contraintes à respecter :

1. Le niveau :

  • Vous êtes libres pour la partie créative, mais le niveau ne devra pas être très long.
  • Il doit comporter une zone de départ et une zone d'arrivée.
  • Des ennemis patrouillants devront être disposés dans le niveau.
  • Des diamants seront positionnés le long du chemin.

2. Le personnage :

  • Il devra pouvoir se déplacer dans toutes les directions et sauter.
  • Il pourra se débarrasser des ennemis en sautant dessus.
  • Il pourra ramasser les diamants.
  • Le jeu s'arrêtera dès que le personnage atteindra la zone d'arrivée.

3. Les ennemis :

  • Ils seront assez petits pour que le personnage puisse leur sauter dessus pour s'en débarrasser.
  • Ils patrouilleront dans une zone et se rueront vers le personnage dès qu'ils le verront.
  • Au contact avec le personnage, celui-ci perdra une vie.

4. L'interface utilisateur (UI) :

  • La vie du personnage sera représentée par 3 cœurs affichés dans un coin de l'écran.
  • Un score tiendra compte du nombre de diamants collectés.
  • Le jeu pourra être mis en pause.

Attention :

  • L'esthétisme n'est pas prioritaire.
  • Les groupes seront constitués de 3 étudiants.
  • Les étudiants souhaitant se consacrer à la partie VR peuvent le faire.
  • La notation sera focalisée sur la fonctionnalité du jeu et la qualité du rapport.

Ma participation sur ce projet :

Les monstres : Ils ont été créés avec des déplacements basés sur un système de navigation (MeshNav). En mode passif, ils patrouillent, mais dès qu'ils détectent le joueur, ils se mettent à le poursuivre. Il est également possible de les éliminer en sautant dessus.

La barre de vie du joueur : Elle reflète visuellement l'état de santé du personnage à l'écran.

Les collectibles : Des diamants ont été répartis dans le niveau, et le joueur peut les ramasser.

L'interface utilisateur : Elle affiche à la fois la vie du joueur (sous forme de cœurs) et le nombre de diamants collectés.

Effet de recul : Lorsqu'un ennemi est éliminé, un léger mouvement de recul est appliqué au joueur pour renforcer le ressenti d'impact.

Décor 3D personnalisé : Un palmier a été modélisé à l'aide de Blender pour enrichir le décor, ainsi qu'un panneau.

Level design : Un niveau de test a été conçu pour mettre en œuvre et tester l'ensemble des mécaniques.

Menu et navigation entre les scènes : Un menu a été intégré pour accéder au jeu, avec un système de liaison entre les différentes scènes (si cette fonctionnalité est pleinement fonctionnelle).
